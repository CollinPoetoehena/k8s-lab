# Kubernetes Worker Role

Configures Kubernetes worker nodes that run application workloads, joining them to the cluster using the token generated by the control plane.

## Description

This role prepares worker nodes and joins them to the Kubernetes cluster using `kubeadm join`. Worker nodes run the kubelet and kube-proxy, handling pod scheduling and execution of containerized applications.

## Responsibilities

- **Cluster Join**: Join worker nodes to the Kubernetes cluster using kubeadm
- **Kubelet Configuration**: Configure kubelet service with appropriate parameters
- **Kube-proxy Setup**: Configure kube-proxy for service networking (if not using Cilium)
- **Node Labels**: Apply custom labels for workload scheduling
- **Resource Allocation**: Configure resource reservation for system services
- **Node Taints**: Apply taints if specific workload isolation is needed
- **CNI Configuration**: Ensure CNI plugin is properly configured for pod networking
- **Storage Provisioning**: Configure local storage or CSI drivers for persistent volumes
- **Monitoring**: Install node monitoring agents (node-exporter, cAdvisor)
- **Health Checks**: Verify node is Ready and can schedule pods
